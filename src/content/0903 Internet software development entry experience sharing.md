同学们好，我先做个简单的自我介绍，我叫吴奔道，也是理学院光电专业毕业的学生， 20 年本科毕业于杭电，目前在互联网公司任前端开发工程师。这次很荣幸受陈许敏老师的邀请来给大家分享我的互联网开发入坑经历。陈老师也是我的本科毕业设计的指导老师，陈老师是我大学 4 年里碰到的为数不多的好老师，对学生认真负责。


我入互联网开发可能得从我大一开始说起。


我记得我当时报考杭电，觉得杭电是理工氛围很好的学校，当时看名字想 “光电信息科学与工程” 里面有 ”信息科学与工程“ 几个字，应该和计算机相关，看起来一切都很美好，于是就来了杭电上大学没想到大学我学了四年的物理，第一学年学的是普通物理。第二第三学年学了电动力学，量子理学，信号与系统，信息光学。。。。虽然大学四年做为一个学渣被物理折磨，不过好在大学四年给予我们大把大把的空余时间，可以把自己的时间投入到自己喜欢的方向。我很庆幸的是我大一刚来大学，参加学校的百团大战（社团招新活动）加入了学校的网络工作室社团——红色家园网络工作室。在这个社团我度过了快乐的四年。和兴趣相投的同学们一起参与建设学校的教务信息化，我们社团当时维护着学校的 bbs 论坛，抢课软件，hduin 校园服务公众号，学分核对系统等等。


在大学二年级的时候，我有想过毕业了从事软件开发工作，不过还比较迷茫，当时还没想好做哪个方向的开发，不过那时我边学边玩，时不时和大四的学长们聊天，了解他们的毕业动向作为参考。我当时对自己非科班出身的情况是有清晰的认识的，知道如果毕业了从事软件开发行业是需要和科班的同学进行竞争的。我是处于弱势的，大二的时候我就找计算机专业的同学了解他们的培养计划，确定我需要补哪些课程，然后断断续续的自学起来。


到了大三的时候，我发现周围的同学都在准备考研，我陷入了犹豫，犹豫要找工作还是和他们一起考研。在大三上学期年末的时候，被周围的同学给同化了，也加入了考研的大军，既然考研那就选择计算机研究生啦，然后我准备了一年，最后没考上，然后就屁颠屁颠的去找工作了。


20 年的时候我大四，眼看着快毕业了，考研又失败了，错过了秋招，没有实习经历，那年疫情还爆发了，当时觉得很糟糕，不过还是硬着头皮找工作了，边刷题边海投简历，不管大小公司，有机会就试。


在投简历的时候我回顾了我过去四年的大学经历，综合分析，我觉得我当时的能力和技能点只能契合前端开发的岗位。边找工作面试边开始补习前端基础知识，在找工作期间，考研准备的专业课给了我很大的帮助，使得我能少补很多计算机基础知识，能更加专注前端专业知识的补充。最后很幸运在快毕业的最后一个月拿到了个比较满意的 offer。正式进入了互联网开发行业。


这是我的大学经历，通过课余时间把精力投入到计算机课程的学习，定期了解行业动向，通过空闲时间的学习来弥补和科班同学的差距，让自己符合一名软件开发工程师的标准，这样子找工作才有底气。


说实话，大学的物理专业学习，我觉得还是有收获的，例如量子力学的课程学习，我对第一性原理有了更好的理解，知道了原来可以通过薛定谔方程计算电子的结构。这种对事物本质的追求，从基础理论方程出发反推计算得出表象的结论，在物理学中应该是很重要的方法之一。你们懂得比我多，我现在物理基本都忘光了，我就不卖弄了，哈哈。第一性原理的思想也对我学习计算机课程有着潜移默化的影响，在学习新技术我都会尝试去了解实现方式和背后的理论知识。计算机技术发展很快，我们只要通过构建扎实的基础知识，将知识点进行相互连结，构建完备的知识体系，其实会发现当下流行的很多技术热点本质都是已经有发展十几年甚至几十年的理论基础支撑的。


讲述完了我的入行经历，接下来我想简单给大家介绍一下互联网开发行业的目前状况以及做为一个合格的初级工程师需要具有怎样的技能储备。


后面的介绍也希望对大家了解互联网开发有一定的帮助，虽然大家不一定将来会进入互联网开发行业，但是基本的计算机编程能力，希望大家还是要具备的，可以做为大家在研究生期间进行科研的有利工具，辅助物理专业的学科研究应该是有意义的。


成为一个合格的初级工程师，首先是计算机基础素养，基础知识要扎实，四门基础课程得熟悉。

- 编程语言，针对不同的方向来选择编程语言，建议最好一门动态语言 + 静态语言
	- 动态语言 python javascript lua ruby

		建议大家入门可以学习 python，python 有很多的科学计算库和机器学习相关的工具库，学习 python 对大家在做科学研究也是有一定帮助的，对一些重计算的工作，都可以通过编程来提升科研效率。

	- 静态语言 c c++ java rust go
- 数据结构与算法 （ 写程序应用常常都会碰到，这是基本课程，不需要精通算法，但是基础的数据结构大家还是要熟练运用）
- 计算机网络 （数据通信交互会经常碰到网络知识，故这个也很重要，大家需要深入了解计算机网络 iso 模型的七层模型，对 web 网络协议深刻的理解）
- 计算机操作系统 （ 如果涉及到系统层面的开发工作，如后端开发工作，操作系统也是很重要的一门课程，通过学习操作系统可以深入了解计算机系统的构造，知道程序是如何跑在计算机上的，在系统软件执行性能调优经常会用到操作系统的知识）
- 计算机组成原理 （了解计算机的构造，知道计算机各个组件是如何有机的结合起来，对写出高效的程序是有很好的帮助的）
- [MIT 6.NULL](https://github.com/conanhujinming/comments-for-awesome-courses/tree/main/%E5%85%B6%E4%BB%96%E8%AF%BE%E7%A8%8B/MIT6.NULL%E5%AE%9E%E7%94%A8%E5%B7%A5%E5%85%B7%E4%BB%8B%E7%BB%8D) （ CS 实用工具，git，terminal， vim 等基础工具）
- 扩展
	- 数据库原理
	- 编译原理
	- 设计模式
	- 分布式系统

四门基础课程在不同的工作方向上都有不同的侧重，做为入门学科，数据结构与算法是每个工程师都必须掌握并熟悉的。针对不同的工程开发岗位可以侧重学习剩下的三门学科。

- 前端工程师

	前端工程师除了学习专业知识，html 和 css 以及 javascript 以外，还需要掌握数据结构与算法（leetcode 解题能力达到能写出 middle 的题目就可以了），计算机网络，浏览器工作原理等，学会使用前端框架 react 或者 vue 等，对前端工程化有了解和运用。要求再高点，还需要会一定的后端开发能力，这时候就需要学会数据库原理，会使用 nodejs 进行后端开发等等。

- 后端/运维工程师

	后端工程师就必须要掌握四门基础课程了，一个都不能落下，且还要学习数据库，分布式系统等课程，至少需要熟练运用一门静态语言，可以是 c++ 或者 java 再或者 go，数据结构与算法能力达到要能写出 leetcode hard 的题目的能力。对 linux 熟练使用等等。

- 测试工程师

	测试工程师要求会低点，但是学的也比较杂，需要有编程技能，对前后端都得有了解，且能设计自动化测试方案，对软件生命周期内的质量管控能提出并落实合适的方案，提升软件交付质量等等。

- 客户端工程师

	目前客户端工程师主要还是安卓和 IOS 工程师，还有部分桌面软件开发工程师，例如 windows，mac 和 linux 客户端开发。客户端开发的能力模型还是得根据不同的客户端进行确定。目前客户端开发工作，基本都是客户端开发工程师和前端开发工程师合作完成，客户端的开发工作，我觉得是前后端能力的结合，即需要前端部分的 UI 实现，也需要对客户端的宿主系统有深入认识，才能做到更好的性能调优和系统定制能力。目前客户端工程师除了互联网 app 有需求，国内安卓定制系统也会有这部分工程师的需求。


上面讲到的是工程开发岗位的划分，下面这个是算法岗位。

- 算法工程师（这个岗位要求专业素养比较高，不建议非科班同学选择，竞争激烈）

	算法工程师要求比较高，基本都要求科班硕士起步，或者本科做过计算机方向的科研项目，大公司基本要求校招的算法工程师都得需要发表过顶会论文。目前算法工程师主要做的方向有推荐系统，自然语言处理，计算机视觉等等，算法工程师主要还是做算法设计和建模已经模型的训练。


上面的划分并不是表示不同的工种不能横跨，只是说明了不同工作方向的侧重点不同。如果入了互联网开发，我还是希望大家能把自己定义为软件开发工程师，不要局限于任何一个工程开发方向。能多学习不同开发方向的知识，不要给自己设限。入门开发可以选择一个方向去努力，等一个方向能力深度够了后就可以考虑横向发展，扩宽能力技能点，让自己发展成为 T 形人才。


校招软件开发岗位，面试官还是希望能看到同学们对开发岗位的热情，和扎实的专业基础，对技术深度要求不会太高。如果能力达标，还是有机会的。


再补充一个工作岗位：硬件开发工程师


这个岗位是这两年互联网公司也投入资源的工作岗位，前几年主要在工业自动化，智能设备制造，物联网公司才会有这一岗位。目前互联网公司的硬件工程师主要还是从事自身业务相关智能硬件开发，这两年很多互联网公司投入到芯片设计，就涌现了一部分芯片设计工程师的人才需求。也和新能源发展起来也有关系，这两年新能源汽车很火，硬件开发工程师和电池材料工程师的需求也很大。


硬件开发工程师主要还是偏电子方向研发，更多需要有编程技能的。如果大家对这个方向感兴趣，我觉得编程能力还是能补齐的，前提是肯下苦功夫。和物理专业勉强沾点边的可能就是硬件方向的工程师（开发方向）。


大家可以在空闲时间多去了解不同公司的招聘信息，来了解不同开发岗位对应聘者的要求，这样子才能知道自己希望从事的方向对人才的技能要求，从而有针对性的去学习提升自己。无论是想从事和自己专业契合度高的工作，还是从事自己感兴趣的工作，都需要具有希望去的工作岗位所要求的能力模型。只有知己知彼，准备上更有目的性。


希望我今天的分享能让大家对互联网开发有个基本了解，供大家对将来职业选择提供点信息参考。

QA：面试中应避免的错误？


这个对校招生来说，应该做到几点

- 面试的时候保持诚实，表现出对应聘岗位的热情，能和面试官有较好的互动。
- 应聘岗位对应的能力得过关，能有清晰阐述项目经历（可以考虑 S（背景）T（任务）A（行动）R（结果） 法则去阐述）。
- 准时面试，不爽约，和面试官保持互相的尊重。

这些都是加分的。还有一个面试小技巧，就是面试完成后，都可以反问面试官对你的评价，如果负责的面试官，会给你分析你的问题点和不足，这样子就能知道自己有哪些不足，面完后可以有针对性的复习，查缺补漏。
